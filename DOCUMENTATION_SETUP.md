# IOsonata Documentation Setup Guide

This guide explains how to deploy the IOsonata documentation to GitHub Pages.

## 📋 What's Included

This documentation infrastructure provides:

- ✅ **Auto-generated API documentation** from Doxygen comments
- ✅ **Hand-written guides** for getting started and architecture
- ✅ **Example browser** with learning paths
- ✅ **Searchable interface** with dark/light themes
- ✅ **Automatic deployment** via GitHub Actions
- ✅ **Mobile-responsive** design

## 🚀 Deployment Steps

### Step 1: Update Doxyfile

Apply the changes from `docs/Doxyfile.patch`:

```bash
cd docs
```

Open `Doxyfile` and make these key changes:

```doxyfile
# Line 57
PROJECT_BRIEF = "Cross-platform C++ HAL for embedded systems"

# Line 952
INPUT = ../include \
        ../src \
        ../ARM/include \
        ../ARM/src

# Line 992
FILE_PATTERNS = *.h *.hpp *.cpp *.c

# Line 1088
EXAMPLE_PATH = ../exemples \
               ../ARM/Nordic/nRF52/nRF52832/exemples

# Line 1334
HTML_OUTPUT = api

# Line 1415
JAVADOC_AUTOBRIEF = YES

# Line 2554
HAVE_DOT = YES

# Around line 1010 (add)
EXCLUDE_PATTERNS = */external/* \
                   */nrfx/mdk/* \
                   */CMSIS/Core/*
```

### Step 2: Enable GitHub Pages

Repository maintainer must:

1. Go to **Repository Settings** → **Pages**
2. Under "Build and deployment":
   - Source: **Deploy from a branch**
   - Branch: **gh-pages** / `/ (root)`
3. Click **Save**

### Step 3: Push to GitHub

```bash
git add .
git commit -m "docs: add comprehensive documentation infrastructure"
git push origin master
```

### Step 4: Watch the Build

1. Go to **Actions** tab on GitHub
2. Watch "Deploy Documentation to GitHub Pages" workflow
3. First run takes ~5 minutes (installs dependencies)
4. Subsequent runs take ~2 minutes

### Step 5: Access Documentation

Once deployed, visit:
```
https://iosonata.github.io/IOsonata/
```

## 🛠️ Local Development

### Install Dependencies

```bash
pip install mkdocs-material
pip install mkdoxy
pip install pymdown-extensions
```

On macOS/Linux, also install Doxygen:
```bash
# macOS
brew install doxygen graphviz

# Ubuntu/Debian
sudo apt-get install doxygen graphviz
```

### Generate Doxygen

```bash
cd docs
doxygen Doxyfile
```

This creates `docs/api/` with HTML documentation.

### Build MkDocs Site

```bash
mkdocs build
```

This creates `site/` with the full website.

### Serve Locally

```bash
mkdocs serve
```

Visit `http://127.0.0.1:8000/` to preview.

**Live reload**: Changes to `.md` files auto-refresh the browser!

### Clean Build

```bash
rm -rf site docs/api
mkdocs build --clean
```

## 📝 Writing Documentation

### Create New Page

1. Create markdown file in appropriate directory:
   ```bash
   # Example: new API page
   touch docs/api/timer.md
   ```

2. Add to `mkdocs.yml` navigation:
   ```yaml
   nav:
     - API Reference:
       - Timer: api/timer.md
   ```

3. Write content using markdown + Material extensions

### Documentation Style Guide

#### Use Admonitions

```markdown
!!! tip "Pro Tip"
    This is a helpful tip!

!!! warning "Important"
    Pay attention to this.

!!! failure "Common Error"
    Here's what went wrong.
```

Renders as colored callout boxes.

#### Use Tabs

```markdown
=== "C++"
    ```cpp
    UART uart;
    uart.Init(cfg);
    ```

=== "C"
    ```c
    UARTDev_t uart;
    UARTInit(&uart, &cfg);
    ```
```

Creates tabbed code blocks.

#### Use Grid Cards

```markdown
<div class="grid cards" markdown>

-   **Getting Started**

    ---

    Quick start guide

    [:octicons-arrow-right-24: Start](link)

-   **API Reference**

    ---

    Complete API docs

    [:octicons-arrow-right-24: Browse](link)

</div>
```

Creates visual card layout.

### Adding API Documentation

For comprehensive API docs, add Doxygen comments to headers:

```cpp
/**
 * @class UART
 * @brief Universal Asynchronous Receiver/Transmitter
 *
 * @implements DeviceIntrf
 *
 * Provides serial communication with DMA support and
 * hardware flow control.
 *
 * @example
 * @code
 * UART uart;
 * uart.Init(config);
 * uart.printf("Hello!\n");
 * @endcode
 */
class UART : public DeviceIntrf {
public:
    /**
     * @brief Initialize UART peripheral
     *
     * @param cfg Configuration structure
     * @return true on success, false on error
     *
     * @note Buffers must be in RAM on Nordic nRF5x
     * @warning Do not call from ISR
     *
     * @see UARTCfg_t
     */
    bool Init(const UARTCfg_t& cfg);
};
```

## 🔧 Troubleshooting

### Build Fails Locally

**Error**: `mkdocs: command not found`

**Fix**:
```bash
pip install mkdocs-material mkdoxy
```

---

**Error**: `doxygen: command not found`

**Fix**: Install Doxygen (see Local Development above)

---

**Error**: `ModuleNotFoundError: No module named 'pymdownx'`

**Fix**:
```bash
pip install pymdown-extensions
```

### GitHub Actions Fails

**Error**: "Documentation build failed"

**Check**:
1. View Actions tab → click failed run
2. Expand logs to see error
3. Common issues:
   - Missing markdown files referenced in `mkdocs.yml`
   - Doxygen syntax errors in comments
   - Invalid YAML in `mkdocs.yml`

---

**Error**: "403 Permission denied"

**Fix**: Enable write permissions:
1. Repository Settings → Actions → General
2. Scroll to "Workflow permissions"
3. Select "Read and write permissions"
4. Save

### Documentation Not Updating

**Problem**: Changes pushed but site doesn't update

**Fix**:
1. Check Actions tab - build must complete successfully
2. GitHub Pages can take 1-2 minutes after build
3. Hard refresh browser: Ctrl+Shift+R (Cmd+Shift+R on Mac)
4. Check `gh-pages` branch exists and has commits

### Links Broken

**Problem**: Internal links show 404

**Fix**: Use relative paths:
```markdown
<!-- ✅ Good -->
[Quick Start](getting-started/quick-start.md)

<!-- ❌ Bad -->
[Quick Start](/getting-started/quick-start.md)
```
